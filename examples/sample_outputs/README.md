# Sample outputs

These are **real outputs** generated by `content_repurposer.pipe` running on the local RocketRide engine, with the source post `examples/sample_blog_post.md` sent through GPT-4o.

To regenerate them yourself, install the RocketRide engine, set `ROCKETRIDE_OPENAI_KEY` in your `.env`, and run `python run_pipe.py` (see the main [README](../../README.md) for the full setup).

| File | Platform | Sample length |
|---|---|---|
| [`linkedin.md`](linkedin.md) | LinkedIn post (≤1,300 chars, hook + insight + soft question) | ~995 chars |
| [`x_thread.md`](x_thread.md) | X (Twitter) thread (5–8 numbered tweets, ≤280 chars each) | ~915 chars |
| [`devto.md`](devto.md) | dev.to article in markdown (TL;DR + 3 takeaways + CTA) | ~1,065 chars |
| [`instagram.md`](instagram.md) | Instagram caption (800–1,500 chars, conversational, with hashtags) | ~1,040 chars |

These outputs vary slightly each time you regenerate them — that's GPT-4o's nondeterminism, not a bug. The shape and platform-specific format (tweet numbering, dev.to markdown structure, Instagram emoji + hashtags) is enforced by the four `agent_rocketride` nodes' system prompts in the `.pipe` file.
