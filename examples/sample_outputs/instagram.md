Spent the weekend building an AI tool to do the most boring part of my job for me 🛠️

Every time I publish a long-form post, I have to retype the same idea four different ways — LinkedIn for the professional take, X for the punchy version, dev.to for the technical writeup, Instagram for the human one.

It's mechanical work. It eats hours. And the rewrites are always slightly inconsistent because by the fourth platform, I'm tired.

So I built a content repurposer using RocketRide — an open-source tool for AI workflows. The whole thing is one JSON file. Ten components. The four rewrites run in parallel. I paste the original post in one box, four platform-specific drafts come out the other side.

The thing that surprised me most wasn't how fast it was. It was how much the tool stopped me from making mistakes I would have made in a hand-rolled version. The connections between components are typed — you literally cannot wire the wrong output into the wrong input. The category of bug where you spend two hours debugging a silent failure just doesn't exist.

This is the kind of side-project that pays for itself in a week. The next version wires it to my publishing flow so the rewrites happen the moment I hit Publish. That's a one-line change in the pipe file.
